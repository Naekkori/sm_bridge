# Rust 프로젝트 규칙

## 1. 언어 및 소통
- 모든 설명과 상호작용은 **한국어**로 진행한다.
- 코드 내 주석은 한국어로 작성하되, 경어체는 사용하지 않는다 (예: "~함", "~임").

## 2. 코딩 스타일 (Rust)
- **함수명**: `snake_case`를 사용한다. (기존 코드 `src/lib.rs` 규칙 따름)
- **구조체/Enum**: `PascalCase`를 사용한다.
- **변수명**: `snake_case`를 사용한다.
- **상수**: `SCREAMING_SNAKE_CASE`를 사용한다.

## 3. 에러 처리
- `wasm_bindgen`을 사용하는 함수는 오류 발생 시 `Result<T, JsValue>`를 반환하여 JS 쪽에서 예외를 처리할 수 있게 한다.
- `unwrap()` 보다는 명시적인 에러 처리를 선호한다 (단, 프로토타이핑이나 확실한 경우 제외).

## 4. 라이브러리 및 의존성
- `wasm_bindgen`, `web-sys` 등을 적극 활용하여 웹 환경과 상호작용한다.
- 불필요한 의존성 추가는 지양한다.

## 5. 기타
- 사용자는 수학에 약하므로, 복잡한 수식이나 알고리즘 설명 시 직관적인 예시를 든다.
- 사용자가 직접 코드를 작성하는 것을 선호하므로, 전체 코드를 덮어쓰기보다는 **가이드와 핵심 로직 위주**로 설명한다.
